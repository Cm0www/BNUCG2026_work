# 计算机图形学实验五：光线追踪（Work05）

## 一、实验内容

本项目使用 **Taichi** 实现 Whitted-Style 光线追踪器。程序采用适配 GPU 并行计算的**迭代式**光线弹射逻辑，不使用递归；每个像素独立追踪主光线及其后续反射/折射光线。

已完成必做内容与两项选做内容：玻璃材质折射、MSAA 抗锯齿。

## 二、已完成功能

### 1. 必做内容

- 搭建三维场景：
  - 无限大棋盘格地面：`y = -1.0`，法线为 `(0, 1, 0)`；根据交点 `x、z` 坐标整数部分的奇偶性生成黑白格纹理。
  - 左侧红色漫反射球：球心 `(-1.5, 0.0, 0.0)`，半径 `1.0`。
  - 右侧银色理想镜面球：球心 `(1.5, 0.0, 0.0)`，半径 `1.0`。
- 迭代式 Whitted 光线追踪：
  - 使用 `throughput` 记录光线传播过程中的能量衰减，初始值为 `(1, 1, 1)`。
  - 使用 `final_color` 累积最终颜色，初始值为 `(0, 0, 0)`。
  - 镜面命中后根据反射公式更新光线方向，反射率为 `0.8`。
  - 漫反射命中后按 Phong 光照模型计算颜色，累积后结束该路径。
  - 最大弹射次数可设置为 `1~5`，默认 `3`。
- 硬阴影：
  - 从漫反射交点向光源发射暗影射线。
  - 若暗影射线在抵达光源前与其他物体相交，则该点只保留环境光。
- 浮点精度处理（Shadow Acne）：
  - 反射光线与阴影射线均使用 `P_new = P + N × 1e-4` 作为新起点，避免自相交黑点。
- UI 交互面板：
  - `Light X / Light Y / Light Z`：实时修改点光源位置并观察阴影变化。
  - `Max Bounces`：实时调整最大弹射次数。

### 2. 选做内容

- 玻璃材质（+15%）：
  - 勾选 `Glass sphere` 后，左侧红球切换为玻璃球。
  - 按斯涅尔定律计算折射光线方向，折射率设为 `1.50`。
  - 当折射条件不成立时自动执行全反射。
  - 折射射线使用相反方向的法线偏移，避免重新命中同一表面。
- MSAA 抗锯齿（+10%）：
  - 通过 `MSAA samples` 设置每个像素的随机主光线采样数，范围为 `1~8`。
  - 对同一像素的多次追踪结果取平均，平滑物体轮廓和棋盘格远处边缘。

## 三、关键原理

### 1. 理想镜面反射

\[
\mathbf{R} = \mathbf{L}_{in} - 2(\mathbf{L}_{in} \cdot \mathbf{N})\mathbf{N}
\]

其中，`L_in` 为入射光线方向，`N` 为与入射光线相反的表面法线，`R` 为反射方向。

### 2. 迭代式路径追踪

传统 Whitted 光线追踪通常通过递归产生反射光线。本项目在 GPU Kernel 中使用固定上限的循环替代递归：镜面命中时更新光线起点、方向和 `throughput`，漫反射命中时将局部光照结果乘以 `throughput` 后加入 `final_color` 并结束循环。

### 3. 硬阴影与 Shadow Acne

从交点向光源发出阴影射线，若途中被遮挡则不计算直接光照。由于浮点误差，射线起点若直接设为交点会和自身表面再次相交，因此将起点沿法线平移一个很小的 `epsilon=1e-4`。

### 4. 玻璃折射与全反射

玻璃球根据光线入射/离开介质的方向设置折射率比 `η = η_i / η_t`，再依据斯涅尔定律计算折射方向。当计算出的透射角不存在时，说明发生全反射，此时转为镜面反射。

### 5. MSAA

对于每个像素，在像素内部随机选取多个子像素位置发射主光线，对追踪结果求平均。采样数增加后，轮廓边缘会更加平滑，但渲染耗时也会相应提高。

## 四、运行环境

- Python 3.10 或更高版本
- Taichi 1.6 或更高版本
- NumPy

安装依赖：

```bash
pip install -r requirements.txt
```

## 五、运行方法

进入 `work05` 文件夹后执行：

```bash
python main.py
```

程序会打开交互窗口。右上角 `Controls` 面板中可直接拖动光源位置、修改最大弹射次数，并启用玻璃球与 MSAA。

如 Mac 上无法正常调用 GPU，可强制使用 CPU：

```bash
python main.py --cpu
```

启动时直接进入选做效果（玻璃球、4 倍 MSAA、5 次弹射）：

```bash
python main.py --glass --spp 4 --max-bounces 5
```

本程序**不会自动生成 PNG**。实验效果使用系统录屏录制后，再转换为 GIF 提交。

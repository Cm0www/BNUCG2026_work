# 计算机图形学实验七：质点-弹簧模型

本实验基于 Taichi 实现三维质点-弹簧布料模拟。布料被离散为规则网格上的质点，质点之间通过弹簧连接，并在重力、阻尼力和弹簧力的共同作用下运动。

基础部分实现了显式欧拉、半隐式欧拉和近似隐式欧拉三种积分方法，以及 GGUI 交互界面。选做部分在结构弹簧的基础上加入剪切弹簧、弯曲弹簧和球体碰撞，使布料的形变和交互效果更完整。

```bash
python3 mass_spring.py --headless --steps 240 --method semi
python3 mass_spring.py --headless --steps 240 --method implicit
```

当终端输出 `headless check passed` 时，说明对应积分方法能够正常运行。

## 4. 已实现内容

### 4.1 基础部分

- 使用 `20 × 20` 个质点构建布料网格；
- 用结构弹簧连接横向、纵向相邻质点；
- 计算重力、阻尼力和基于胡克定律的弹簧力；
- 使用 `ti.atomic_add` 累加多个弹簧对同一质点的作用力，避免 GPU 并行写入冲突；
- 通过速度钳制限制质点最大速度，降低数值爆炸的风险；
- 实现显式欧拉、半隐式欧拉和近似隐式欧拉三种积分方法；
- 使用 Taichi GGUI 完成三维渲染，并支持积分器切换、阻尼调节、暂停和重置。

- 在结构弹簧基础上加入剪切弹簧；
- 加入弯曲弹簧，提高布料的抗弯曲能力；
- 在场景中加入球体；
- 对进入球体内部的质点进行位置和速度修正，实现简单的布料-球体碰撞；
- 在控制面板中支持开启或关闭剪切弹簧、弯曲弹簧和球体碰撞。

## 5. 实现思路

### 5.1 质点-弹簧模型

布料中的每个质点保存位置、速度和受力信息。每一帧中，程序首先对质点施加重力和阻尼力，再遍历全部弹簧并计算弹簧力。弹簧力由胡克定律得到：弹簧被拉长时会产生拉力，被压缩时会产生推力，使两端质点尽量恢复到弹簧原长。

因为一个质点通常连接多根弹簧，多个线程可能同时更新同一质点的受力，所以程序使用 `ti.atomic_add` 对受力进行安全累加。

### 5.2 三种积分方法

- **显式欧拉**：使用当前速度更新位置，再使用当前加速度更新速度。实现简单，但对时间步和弹簧刚度更敏感。
- **半隐式欧拉**：先更新速度，再使用新速度更新位置。相比显式欧拉，稳定性更好。
- **近似隐式欧拉**：使用固定点迭代估计下一时刻状态，稳定性更强，但会带来更明显的数值耗散。

### 5.3 剪切弹簧、弯曲弹簧与球体碰撞

剪切弹簧连接网格对角线方向的质点，能够减少斜向拉伸和扭曲；弯曲弹簧连接相隔一个网格的质点，可以提高布料的抗弯曲能力。

球体碰撞中，程序每次更新后判断质点到球心的距离。若质点进入球体内部，则把质点投影回球面，同时去除朝球体内部的法向速度分量，避免继续穿透。

## 7. 结果分析

### 7.1 阻尼系数的影响

阻尼为 `1.0` 时，布料在重力作用下下落后会持续振动，能量衰减较慢。阻尼增大到 `5.0` 后，布料的摆动幅度更快减小，能够更快达到稳定状态。这说明阻尼力能够有效消耗系统能量，抑制布料的振荡。

### 7.2 积分方法的差异

显式欧拉计算简单，但在时间步较大或弹簧刚度较高时更容易发生抖动。半隐式欧拉在稳定性和计算开销之间较平衡。近似隐式欧拉稳定性更高，但布料的运动会显得更平缓一些。

### 7.3 选做内容的作用

加入剪切弹簧后，布料对斜向拉伸和扭曲的抵抗能力更强；加入弯曲弹簧后，布料不容易出现过度折叠，整体形态更加自然。球体碰撞使布料能够与障碍物发生接触和形变，增强了场景的物理交互效果。

本实验完成了基于 Taichi 的质点-弹簧布料模拟，并实现了三种数值积分方法。通过调节阻尼系数，可以直观观察系统振动衰减的差异。在基础模型上加入剪切弹簧、弯曲弹簧和球体碰撞后，布料的形态和交互效果更加完整，也加深了对数值积分、物理仿真和GPU并行计算的理解。
